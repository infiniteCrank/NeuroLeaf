This demo uses multiple ELMs to classify the language of a greeting by combining dense character-level features with normalized input metadata.

üîç Current Architecture:

  1. AutoComplete ELM suggests the full greeting from partial input.
  2. CharacterLangEncoderELM encodes the autocompleted greeting into a normalized vector representation.
  3. Metadata like input length, character diversity, vowel ratio, and punctuation ratio is extracted and normalized.
  4. FeatureCombinerELM combines the vector + metadata to classify the language.
  5. If the combiner is uncertain or wrong, RefinerELM corrects low-confidence predictions.

The display includes model source: [C] = Combiner, [R] = Refiner, [AC] = Autocomplete input used.
